# デモ動画 台本

## 概要

- 所要時間: 3〜4分
- テーマ: 「自然言語でロボットに指示を出し、タスクが自動実行される」

---

## シーン 1: 導入 (30秒)

**画面**: README のアーキテクチャ図

**ナレーション**:
> RoboTasker は、自然言語でロボットに指示を出し、タスクの生成・実行・記録・可視化を行う統合システムです。
> Go バックエンド、ROS2、LLM を組み合わせ、業務利用を想定したアーキテクチャで設計しました。

---

## シーン 2: Chat でタスク作成 (60秒)

**画面**: ブラウザで http://localhost:3000 を開く

1. Chat 画面を表示
2. 「会議室Bに資料を届けて」と入力
3. レスポンスを確認
   - LLM 解析結果: action=deliver, target=会議室B, item=資料, confidence=90%
   - タスク作成・ロボット割当の自動処理

**ポイント**:
> 自然言語をLLMで構造化タスクに変換しています。
> アクション・目的地・アイテム・優先度を自動判定し、空いているロボットに割り当てます。

4. 「急ぎで倉庫に荷物を運んで」と入力
   - priority: HIGH が自動判定される
   - 2台目のロボットに割り当て

---

## シーン 3: Tasks で進捗確認 (30秒)

**画面**: Tasks ページに遷移

1. タスク一覧でステータスがリアルタイムに変わる様子を見せる
   - pending → assigned → in_progress → completed
2. 所要時間が記録されていることを確認
3. キャンセルボタンの存在を見せる

---

## シーン 4: Dashboard でロボット可視化 (45秒)

**画面**: Dashboard ページに遷移

1. 2Dマップ上でロボットの位置が表示される
2. 統計カード: 全タスク数、完了数、平均所要時間
3. ロボット詳細: バッテリー残量、現在地、ステータス
4. 新しいタスクを送信し、マップ上でロボットが動く様子（WebSocket リアルタイム更新）

**ポイント**:
> バックエンドの WebSocket を通じて、ROS2 のロボット状態がリアルタイムにフロントエンドに配信されています。

---

## シーン 5: ROS2 ノード構成 (30秒)

**画面**: ターミナルで `make ros2-topics` を実行

1. ROS2 トピック一覧を表示
2. `ros2 topic echo /robot/state` でロボット状態メッセージを見せる

**ポイント**:
> カスタムメッセージ定義（TaskCommand, TaskStatus, RobotState）で
> バックエンドとROS2が型安全に通信しています。

---

## シーン 6: アーキテクチャ説明 (45秒)

**画面**: ARCHITECTURE.md またはスライド

1. レイヤー構成を説明
   - Frontend → Backend API → LLM / ROS2 Bridge → ROS2 Nodes
2. インターフェース設計を説明
   - MockClient / WSClient の切り替え
   - MockLLM / OpenAI API の切り替え
3. 将来の拡張ポイント
   - Gazebo 連携、Nav2 導入、実機接続

---

## シーン 7: 締め (15秒)

**ナレーション**:
> バックエンド設計力、AI 統合力、ROS2 の理解を1つのプロジェクトで示せる構成にしました。
> コードは GitHub で公開しています。ご覧いただきありがとうございました。
